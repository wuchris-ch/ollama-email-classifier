# Dataset

A 954-email synthetic corpus generated by `synthesize.py`. Stdlib only,
deterministic seeds (1 / 2 / 3 for train / dev / test).

## Why synthetic
- Reproducible with no API key and no PII concerns.
- Lets a reader run the full pipeline on a fork without legal worries.
- Trades realism for control; see "limitations" below.

## Label taxonomy

| Label | Description |
|---|---|
| `personal` | Friends, family, social plans |
| `work` | Internal team comms, reviews, planning |
| `finance` | Statements, alerts, tax, banking |
| `promo` | Marketing, sales, newsletters |
| `transactional` | Order receipts, shipping, password resets, 2FA |
| `spam` | Phishing, scams, advance-fee fraud |

## Splits

| Split | Emails | Per class | Seed |
|---|---|---|---|
| train | 600 | 100 | 1 |
| dev | 150 | 25 | 2 |
| test | 204 | 34 | 3 |

## Reproduce
```bash
python data/synthesize.py
```

## Limitations
- No real-world noise (HTML, signatures, threading, multi-language).
- Transactional vs. promo overlap is the most realistic source of confusion;
  spam vs. transactional (phishing-as-receipt) is the most useful one.
- 60 base templates with placeholder fills, so unique surface count is
  bounded; not a stress test of memorization vs. generalization.
